# 易拍选 迭代需求文档 V1.19

**文档编号**: EPX-PRD-009  
**基于版本**: V1.18  
**迭代名称**: "经营深化" — 财务闭环、消息触达与灵活交付  
**预计周期**: 3-4周  
**产品经理**: 待定  
**交付对象**: 开发团队  

---

## 一、迭代背景

V1.18 已完整覆盖工作室"审片协作-项目管理-财务追踪"三大核心链路。使用者在最终审阅后，提出四个深化方向：

- 财务闭环：从"记录收入"升级为"核算利润"，需要增加支出记录。
- 消息触达：客户端被动刷新升级为主动通知，需要微信模板消息。
- 灵活交付：交付包内容支持按需勾选，满足不同客户要求。
- 批量管理：项目看板支持批量操作，提升多项目并行管理效率。

以上功能开发成本可控，风险较低，建议作为 V1.18 发布后的下一个迭代版本。

---

## 二、功能需求

### F-54 财务支出记录与利润核算 [P1]

**背景**

V1.18 已实现合同金额录入和分阶段付款汇总。但老板还需要记录项目支出，才能核算实际利润。目前支出数据在系统外管理，财务视图不完整。

**F-54-1 支出记录录入**

- 入口：项目详情页 → "更多"菜单 → "支出记录"。
- 支出字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 支出类别 | 下拉选择 | 是 | 修图师工资 / 外包费用 / 场地租赁 / 道具采购 / 交通差旅 / 其他 |
| 金额 | 数字输入 | 是 | 单位：元，支持2位小数 |
| 支出日期 | 日期选择 | 是 | 默认当天 |
| 备注 | 文本输入 | 否 | 支出说明 |

- 支出列表：按时间倒序展示所有支出记录，支持编辑和删除。
- 支出合计：列表底部显示支出总金额。

**F-54-2 利润核算**

- 利润计算：利润 = 合同总金额 - 支出合计。
- 利润率：利润率 = 利润 ÷ 合同总金额 × 100%。
- 显示位置：
  - 项目详情页顶部信息栏增加"预期利润"字段。
  - 战情室财务面板增加"利润汇总"行。
- 数据导出：战情室财务导出 Excel 增加支出和利润列。

**验收标准**

- 支出记录可正常增删改查。
- 利润计算准确，合同金额变更或支出变更后自动重算。
- 财务导出包含完整收支和利润数据。

### F-55 微信模板消息通知 [P2]

**背景**

目前客户需要刷新页面才能看到修图师处理状态。如果能通过微信模板消息主动提醒，客户体验将显著提升，减少"不知道改好了没"的等待焦虑。

**F-55-1 触发场景**

| 触发事件 | 模板消息内容 |
|---------|-------------|
| 修图师完成所有修图 | "您的审片意见已全部处理完成，点击查看确认。" |
| 修图师发起追问 | "修图师对【图片名】的修改意见有疑问，点击回复。" |
| 项目临近截止日期 | "【项目名】即将截止，请尽快确认审片意见。" |
| 老板一键催稿 | "摄影师提醒您确认【项目名】的审片意见。" |

**F-55-2 配置入口**

- 工作空间设置 → "微信通知" Tab。
- 配置项：
  - 微信公众号 AppID
  - 微信公众号 AppSecret
  - 模板消息 ID（每种场景一个）
- 配置后提供"发送测试消息"按钮，验证配置是否生效。

**F-55-3 用户订阅**

- 客户首次打开分享链接时，页面底部弹出"是否接收微信处理通知"的轻提示。
- 客户点击"允许"后，通过微信 JS-SDK 获取订阅授权。
- 客户可在客户端页面设置中随时关闭通知。

**F-55-4 发送限制**

- 同一客户同一项目，同类通知24小时内最多发送1次。
- 避免重复骚扰。

**验收标准**

- 配置正确后，触发事件可收到微信模板消息。
- 消息内容准确，点击可跳转到对应项目。
- 用户可自由开关通知订阅。

### F-56 交付包内容自定义 [P1]

**背景**

V1.18 交付包包含全部内容（成片+Excel+PDF+ZIP）。但不同客户需求不同：有的只要成片，有的还要修改前后对比图，有的要包含巡检报告。目前无法选择，需要手动整理。

**F-56-1 勾选交付内容**

- 入口：项目详情页 → 点击"交付包"按钮 → 弹出内容选择弹窗。
- 勾选项：

| 内容项 | 说明 | 默认 |
|--------|------|------|
| 最终成片 | 所有确稿通过的图片 | ✅ 勾选 |
| 修改前后对比图 | 每张图片的修改前后对比 | ☐ |
| 原片 | 原始拍摄图片 | ☐ |
| 色差巡检报告 | PDF 格式 | ☐ |
| 光影巡检报告 | PDF 格式 | ☐ |
| 交付清单 | Excel 格式 | ✅ 勾选 |
| 项目报告 | PDF 格式 | ✅ 勾选 |

- 勾选后实时显示预计文件大小和图片总数。
- 确认后按勾选内容打包生成 ZIP。

**F-56-2 保存为默认模板**

- 弹窗底部提供"保存为默认配置"复选框。
- 勾选后，下次打开交付包弹窗自动应用该配置。
- 默认配置存储于工作空间级别。

**验收标准**

- 交付包内容按勾选项正确生成。
- 未勾选的内容不出现在 ZIP 中。
- 默认配置保存后下次生效。

### F-57 项目看板批量操作 [P2]

**背景**

项目数量增多后，批量归档、批量修改截止日期等操作需求越来越频繁。目前只能逐个操作，效率低。

**F-57-1 批量选择模式**

- 入口：项目看板右上角增加"批量操作"按钮。
- 交互：
  - 点击进入批量选择模式。
  - 每个项目卡片左上角出现复选框。
  - 顶部显示"已选 N 个项目"和"全选/取消全选"。
  - 再次点击"批量操作"或按 ESC 退出选择模式。

**F-57-2 批量操作项**

| 操作 | 说明 |
|------|------|
| 批量归档 | 将选中项目状态变更为"已归档" |
| 批量修改截止日期 | 统一设置新的截止日期 |
| 批量删除 | 删除选中项目（二次确认） |

- 批量操作后显示结果提示："成功归档 5 个项目，1 个项目因状态为进行中而跳过"。
- 不可操作的项目自动跳过并提示原因。

**验收标准**

- 批量选择流畅，全选/取消全选正确。
- 批量操作结果准确，不可操作项正确跳过。
- 操作日志记录批量操作详情。

## 三、优先级与排期建议

| 优先级 | 功能 | 理由 | 建议排期 |
|--------|------|------|---------|
| P1 | F-56 交付包内容自定义 | 直接影响交付效率和客户满意度 | V1.19 优先 |
| P1 | F-54 财务支出记录与利润核算 | 补完财务闭环，老板强需求 | V1.19 同版 |
| P2 | F-55 微信模板消息通知 | 需微信开放平台配置，有一定接入成本 | V1.19 或后续 |
| P2 | F-57 项目看板批量操作 | 多项目工作室刚需，但非紧急 | V1.19 或后续 |

四个功能均为低耦合独立模块，可并行开发。建议 V1.19 在 V1.18 发布后 3-4 周内完成。

---

## 四、验收总则

1. 支出记录：可正常增删改查，利润计算准确，财务导出完整。
2. 微信通知：配置正确后触发事件可收到模板消息，点击可跳转。
3. 交付包自定义：按勾选项正确生成交付包，默认配置可保存复用。
4. 批量操作：批量选择流畅，操作结果准确，日志记录完整。
5. 兼容性：所有新增功能不影响已有功能的正常运行。

---

## 产品经理附注

V1.19 的核心主题是"经营深化"。V1.18 让工作室老板能"管图管钱"，V1.19 进一步让他能"核算利润、主动触达客户、灵活交付、高效管理多项目"。这四个功能补齐后，产品将从"协作工具"完整进化为"工作室经营平台"。

建议将 F-54 和 F-56 作为本迭代必须交付的核心功能，F-55 和 F-57 可根据微信开放平台接入进度和开发资源灵活排期。
