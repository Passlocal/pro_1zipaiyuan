## 三、用户体验问题（UX-001 ~ UX-002）

### 3.1 UX-001：AI 风格样片预览接口返回 404

| 属性 | 内容 |
|------|------|
| **发现阶段** | 用户试用模拟（修图师角色第6步） |
| **优先级** | P1 |
| **现象** | `POST /ai/style-preview` 传入有效 `imageId` + `style` 参数后，返回 HTTP 404 |
| **影响** | 修图师无法预览 AI 风格效果，功能入口形同虚设 |
| **根因推测** | 后端路由 `/ai/style-preview` 未注册或 AI 服务未部署 |
| **建议修复** | 1. 检查后端路由注册；2. 若 AI 服务为外部依赖，增加 Mock 模式或降级提示；3. 前端增加接口异常时的友好提示（替代静默失败） |
| **关联** | FB-005（第一轮反馈中客户对 AI 风格滑块的好评依赖此功能） |

### 3.2 UX-002：项目归档后，修图师待办任务不可见

| 属性 | 内容 |
|------|------|
| **发现阶段** | 用户试用模拟（修图师角色第2步） |
| **优先级** | P2 |
| **现象** | 在 ux_test.py 中，项目先被归档（步骤14），随后恢复（步骤15），但修图师登录后 `GET /my-tasks` 返回的待办列表可能为空或任务数异常，取决于归档时间点 |
| **影响** | 修图师无法看到已归档项目中的待处理任务，导致任务遗漏 |
| **根因分析** | 归档操作可能将项目状态标记为 `archived`，而 `my-tasks` 接口默认过滤了已归档项目，缺少"包含已归档项目"的查询选项 |
| **建议方案** | 1. `my-tasks` 接口增加 `includeArchived` 参数；2. 或归档前校验是否存在未处理卡片，若有则阻止归档；3. 前端待办列表增加"已归档项目任务"折叠区 |
| **关联** | SUG-002（归档确认提示）、FB-009（批量操作筛选） |

## 四、优化建议（SUG-001 ~ SUG-003）

### 4.1 SUG-001：战情室按客户/品牌分组

| 属性 | 内容 |
|------|------|
| **发现阶段** | 产品经理走查验收 |
| **优先级** | P2 |
| **现状** | 战情室项目列表为平铺展示，未按客户维度分组 |
| **建议** | 老板视角下，战情室项目列表支持按"客户名称"分组折叠，快速查看每个客户的项目进度和状态 |
| **价值** | 老板日常管理中有多个客户同时进行，分组视图可减少信息扫描成本 |
| **关联** | FB-006（战情室预警规则）、FB-007（成员负载） |

### 4.2 SUG-002：归档操作前增加确认提示

| 属性 | 内容 |
|------|------|
| **发现阶段** | 用户试用模拟 + 产品经理走查 |
| **优先级** | P2 |
| **现状** | 归档操作（单个+批量）无二次确认，直接执行 |
| **建议** | 归档前弹出确认对话框，提示：1. 该项目下还有 X 条未处理意见卡片；2. 归档后项目将移至归档区；3. 如需恢复可在归档区操作 |
| **价值** | 防止误操作，尤其是批量归档时避免将未完成项目归档 |
| **关联** | UX-002（归档后任务不可见）、FB-009（批量操作） |

### 4.3 SUG-003：客户标注入口增加引导

| 属性 | 内容 |
|------|------|
| **发现阶段** | 产品经理走查验收 |
| **优先级** | P3 |
| **现状** | 客户通过分享链接打开后，标注入口不够明显，首次使用的客户可能不知道如何标注 |
| **建议** | 1. 首次打开分享链接时，显示引导浮层（Coach Mark）："点击图片即可标注修改意见"；2. 标注工具栏增加文字提示 |
| **价值** | 降低客户学习成本，提升标注功能使用率 |
| **关联** | FB-001（画笔不跟手）、FB-002（意见汇总）、FB-003（移动端热区） |

## 九、产品经理总结

### 质量评估

V1.19 经过三轮测试（全面测试 78 项 + E2E 测试 41 项 + PM 走查 28 项），**147 项全部通过**，核心功能闭环完整，安全漏洞已修复，代码质量达标。

### 本轮成果

1. **3 个缺陷修复**：类型强转崩溃、XSS 注入（后端 + 前端）全部修复并验证通过
2. **API 完善**：付款阶段数据在项目创建和查询接口中均已返回
3. **测试体系建立**：形成了覆盖 8 个维度的自动化测试套件，可在后续迭代中复用

### 遗留风险

1. **UX-001（AI 风格 404）**：影响客户侧的 AI 风格预览功能，建议 V1.19 hotfix 或 V1.20 首修
2. **移动端适配**：两轮反馈中 3 个角色均提及，但目前缺乏移动端专项测试环境
3. **运维/部署**：限流配置（2 req/s）在自动化测试中会产生 429 误报，生产环境需根据实际流量调整

### 建议

**V1.20 聚焦三个主题：**

1. **AI 功能修复**（UX-001）：确保已上线的 AI 风格预览功能可用
2. **移动端体验专项**（FB-003、FB-018、FB-022）：三个角色共同诉求
3. **数据可见性提升**（FB-002、FB-006、FB-012、FB-021）：减少"回头看"成本

**V1.20 预计总工时：约 18 人天**（含 P0 + P1，含联调测试）
