# 易拍选（EpicShot）应急预案 V1.2.0

---

## 1. 服务宕机

### 现象
- 用户无法访问
- Health check 返回非 200

### 处理
1. 检查进程：`lsof -i:3000`
2. 检查磁盘：`df -h`（确保有 > 1GB 可用空间）
3. 检查内存：`free -m`
4. 重启服务：`cd /opt/epicshot && npm start`
5. 通知：运维群 + 产品群

### 升级条件
- 5 分钟内无法恢复 → 通知 CTO
- 15 分钟无法恢复 → 通知用户（公众号/站内通知）

---

## 2. 数据丢失

### 现象
- 用户反馈数据消失
- 数据库查询返回空

### 处理
1. 立即停止服务（防止写入覆盖）
2. 检查软删除回收站：`SELECT * FROM comment_cards WHERE deleted_at IS NOT NULL`
3. 检查 WAL 日志：`ls -la data/epicshot.db-wal`
4. 从最新备份恢复（见 BACKUP_STRATEGY.md）
5. 恢复后验证：运行 `test-backend-deep.sh`

---

## 3. 安全事件

### 现象
- 异常登录 / 大量 401
- 未授权数据访问
- Token 泄露

### 处理
1. 立即轮换 JWT_SECRET
2. 吊销所有活跃 Token（重启服务）
3. 检查访问日志：`grep ERROR /var/log/epicshot.log`
4. 通知受影响用户
5. 修复漏洞后发布补丁

---

## 4. AI 服务故障

### 现象
- AI 端点返回错误
- 样片生成/巡检超时

### 处理
1. 检查 AI 服务状态
2. 系统自动重试（1 次，2s 间隔）
3. 用户端显示明确错误提示 + 重试按钮
4. 如需长时间降级，通知用户：AI 功能暂时不可用，基础标注功能正常

---

## 5. 性能降级

### 现象
- API 响应 > 2s
- 并发请求排队

### 处理
1. 检查并发数：`netstat -an | grep :3000 | wc -l`
2. 临时提高速率限制阈值
3. 必要时横向扩容（负载均衡 + 多实例）
4. 检查是否有慢查询：查看 Sentry Performance
